Repl.it
仕事にも教室でも使えるクラウド共同開発環境

Repl.it は、学生や開発者がアプリケーションを学び、コラボレーションし、構築し、ホストし、デプロイするための、世界初のオンライン開発環境を構築しています。

Google Cloud による成果

  • 開発者や学生の共同学習やデプロイメントに対応するため、必要に応じてスケールアップやスケールダウンが可能である
  • Google Container Registry を使用して、ネットワークを介さずにビルドのプッシュ/プルを迅速かつ安全に行うことができる。
  • 使いやすい Google CLI で開発者と学生の体験を向上させる

月間アクティブユーザー数100万人

Repl.itは、従来は一人で楽しむものと考えられていたものを、クラウド上で多人数で楽しめるようにすることで、ソフトウェア開発や教育に革命をもたらしています。

Repl.it の CEO 兼共同設立者である Amjad Masad は「月間100万人のプログラマーを突破しました」と語ります。プログラミングを身近なものにし、ソフトウェアの開発と共有を容易にする」というたった一つのミッションに人生の大半を捧げてきた Amjad にとって、このマイルストーンは大きな誇りです。

より良いプログラミング体験を提供したいという Amjad の情熱は、数年前、Codecademy の創設エンジニアとして働いていたときに始まりました。Codecademy は、4,500万人のユーザー、特に十分なサービスを受けていないコミュニティのユーザーに、12種類以上の言語による無料のプログラミングレッスンを提供しています。それは、教室だけでなく、商業的な開発者のためにも、これまでにないオンライン開発環境を構築することでした。

同社のサービスは2つのグループに人気があります。学生にとって Repl.it はプラグアンドプレイの教室用リソースです。米国の幼稚園から12年生までの学校の約20％が、コードジャムやハッカソンなどの活動でRepl.itを利用しています。開発者にとって Repl.it は、共同作業が可能なライブワーク環境であり、Web サービスのホスティング、製品のパッケージング、アプリケーションのデプロイなど、より高度な機能を構築し、反復して利用することができます。

ある CPU のしきい値に達すると、Google Cloud Load Balancing は新しいインスタンスをインスタンス化し、必要に応じてアップダウンします。

Repl.it、CEO兼共同設立者、Amjad Masad 氏

通常、60カ国以上、7万人もの開発者が Repl.it を介して共同作業を行っています。彼らは Repl.it のマルチプレイヤー機能を利用して、Repl.it の共有コンテナで作業を行い、Repl.it がサポートする47の言語のいずれかまたは複数の言語でプログラミングを行います。Repl.it は、統合開発環境（IDE）のブラウザに言語サポートを組み込んでいます。

開発者や学生にとって Repl.it は、サードパーティ製パッケージの入手、セットアップ、サポート、デバッグ、IDE への追加にかかる時間を最小限に抑えるという、実質的な価値を提供します。「開発者や学生は数秒でコーディングセッションに参加し、すぐに他の人とコラボレーションしたい」と Repl.it のデザイナーである Haya Abdalla は言います。そのために、ソフトウェアを対話的に書き、遅延なく実行することをサポートするシェルUI の REPL から名前をとったと Haya 氏は付け加えます。

最も注目すべき点は、7人のスタッフで、これだけ多くの開発者にフルサービスを提供していることでしょう。Repl.it のチームは、Google Cloud Platform（GCP）と Google Kubernetes Engine を利用して DevOps チームの効率を高め、プラットフォームの管理と拡張に時間を割いています。

1台のVMからクラウドへ

Repl.it は、Google Cloud for Startups と協力することで、柔軟なパートナー関係と信頼できる技術環境を手に入れました。Amjad と彼のチームは、 Google Cloud と協力して、Repl.it のエコシステムを構成する Redis データベースやコンテナなどのレガシーなオープンソース部品を、スケーラブルなクラウドベースのプラットフォームに統合しました。

Repl.it プラットフォームは、Digital Ocean 上の Droplet という1台の仮想マシンから始まりました。Amjad は、セキュアシェルプロトコル（SSH）とgit cloneを使用してデプロイしました。2 年足らずでユーザーコミュニティが100万人に達した Repl.it の成功は、新たな課題をもたらしました。

当初の課題は実行環境、つまり開発者がコードを安全かつ同時に実行できるようにするコンテナシステムでした。このシステムは急速に成長し、需要に応じて拡張する必要のある、急速に拡大する分散システムへと変化しました。

この規模のインフラを構築、デプロイ、維持するための解決策を見つけることは、Digital Ocean の Droplets 環境を圧倒しました。「自分たちで Redis を管理し、自分たちで Docker コンテナレジストリを管理していました」「スケールアップが必要になったとき、より良いデプロイプロセスに移行しなければなりませんでした。」と Amjad 氏は振り返ります。

「Repl.it は最初に AWS を利用しましたが、組織のニーズを満たすものではありませんでした。ドキュメントは役に立たず、Cloud Deploy は期待通りに機能せず、AWS チームからのサポートも全般的に不足していました」と Amjad 氏は言います。これが GCP への移行のきっかけとなりました。

Google のコマンドラインインターフェースはとてもよくできています。探索しやすいし、ドキュメントも見つけやすい。

Repl.it、CEO兼共同設立者、Amjad Masad 氏

レガシーなオープンソースを拡張するためのクラウド提携

現在、Repl.it は Ansible を使用してデプロイメントをプッシュし、GCP の Container Registry でインスタンステンプレートを作成しています。Cloud Memorystore は、Redis データベース用の低レイテンシーのインメモリーストアを提供します。Compute Engine は、インスタンスグループを自動的にスケーリングします。

Repl.it の実行環境では、約 10,000 個の同時実行可能なコンテナをサポートする必要があるため、これは非常に重要な要素です。一定の CPU 閾値を超えると、Google Cloud Load Balancing が新しいインスタンスをインスタンス化し、必要に応じてアップダウンします。

コード・イン・ザ・ブラウザの環境

Repl.itのGCPとの統合が進むにつれ、Amjadは、クラウドストレージとのコミュニケーションをより迅速に行うための新しいコードインザブラウザのソリューションを見つけました。この技術革新は、 Amjad が以前から持っていたエンジニアリングの専門知識によってもたらされました。

2010年、ヨルダンの Princess Sumaya University でコンピュータサイエンスを専攻していた Amjad は、開発環境を整えるプロセスにうんざりしていましたが、ブラウザにはコーディング作業を効率化する大きな可能性があると確信しました。彼は、最初のオープンソースのブラウザ内 REPL の構築に着手しました。米国のいくつかの企業はすぐにその価値を認め、Codecademy 社もその価値を認めて、教室のプラットフォームに統合しました。

Amjad の新しいクラウドベースのブラウザ環境は、Cloud Storage の署名付き URL の機能を利用しています。Repl.it IDE のブラウザは、新しいファイルを作成する際に、ウェブサーバから署名付き URL を取得して、Cloud Storage にファイルを作成します。このようにして、ブラウザは Cloud Storage に直接書き込み、Cloud Storage から読み取ることができるのです。これにより Repl.it は、Cloud Storage を利用して、ファイルとファイルシステムを同期させることができます。

スタートアップ企業に言いたいのは、スケールアップする前に、Google Cloud Platform が提供するスタートアップ用のプログラミングで独自のアーキテクチャを活用し、Google スタッフのエンジニアと話をすることです。

Repl.it、CEO 兼共同設立者、Amjad Masad 氏

統合の柔軟性、優れたデザインの UI

Repl.it は、Google Cloud の新機能を利用して、Cloud Functions のオンデマンド課金の柔軟性をコンテナの世界にもたらしています。「これでさらにコストを下げることができ、お客様にはここで本番システムを構築し、当社の IDE を介して Google Cloud 上でホストするという柔軟性を提供することができます」とAmjad 氏は言います。

BigQuery は Repl.it のユーザー理解を深めます。Repl.it は毎日 BigQuery を使用しています。BigQuery はレガシーの Segment.com プロキシと連動しており、学生や開発者がどのようにサイトを利用しているかを示す値を取得しています。Amjad 氏は、より多くのイベントをキャプチャし、コンポーネントの使用状況を表示する BigQuery の新しい UI に興奮しています。

「Repl.it は、GCP に搭載された新しいインタラクティブなクラウドコマンドラインインターフェイス（CLI）が、学生や開発者にとって最適だと考えています。実によくデザインされています」と Amjad 氏は説明します。探索しやすく、ドキュメントも見つけやすいです。また、彼は CLI の自動更新機能も気に入っています。Google Cloud の新バージョンがリリースされると、CLI は自動的にユーザーに同期を取るよう促します。

「スタートアップ企業では、スケーラビリティが発生するまで心配してはいけないという考え方があり、その大部分は正しいと言えます。しかし、実際にスケールアップしてしまうと、大変なことになります」と彼は笑いながらそうアドバイスします。スタートアップ企業に言いたいのは、規模を拡大する前に、Google Cloud Platform が提供するスタートアップ用のプログラミングで独自のアーキテクチャを活用し、Google のスタッフエンジニアと話をすることです。

Repl.it について

Repl.it は、オンラインでの共同開発や教室でのコーディングのために、フル機能を備えたサーバーレスのハブを構築しています。サンフランシスコを拠点とし、2016年に設立された Repl.it は、急増する開発者や学生に対応するため、ソリューションをクラウドに移行しました。
